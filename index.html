RULES AND REGULATIONS

Hackathon Rules & Guidelines

PYTHON HACKATHON FRAMEWORK & GUI LIBRARY

  • Tkinter - GUI Library
  • Django - Framework

PYTHON HACKATHON DOMAIN

  • Windows-based
  • Web-based

General Rules

    Eligibility:
  • Open to all I-year students enrolled in the specified academic program.
  • Participants must register before the deadline.

    • Team Composition:
  • Teams of 3-5 members.
  • Teams must be finalized at the start of the event.
  • All members must actively contribute to the project.

    • Code of Conduct:
  • Participants must maintain professionalism and adhere to ethical behavior.
  • Harassment, plagiarism, or any inappropriate behavior will lead to disqualification.

Event Structure

    Hackathon Duration:
  • The event will last 8 hours.
  • Time extensions will not be permitted unless approved by the organizers.

    • Theme & Problem Statements:
  • Problem statements will be disclosed at the start of the event.
  • Teams must choose one problem statement to work on.

    • Deliverables:
    Teams must submit:
  • Source code (on GitHub or as a ZIP file).
  • A short demo video (optional).
  • A README file detailing the project description, features, and usage.

Judging Criteria

    Evaluation Metrics:

  • Functionality (30%): Does the project meet the requirements of the problem statement?
  • Innovation (20%): Is the project creative and unique?
  • UI/UX Design (20%): Is the interface user-friendly and aesthetically pleasing?
  • Code Quality (20%): Is the code structured, modular, and well-documented?
  • Presentation (10%): How effectively did the team demonstrate and explain their project?

    • Judging Panel:
  • Judges will consist of technical experts and mentors.

    • Presentation:
  • Each team will have 5-10 minutes to demonstrate their project and explain its features.

Technical Rules

    Permitted Technologies:
  • Participants must use Python as the primary programming language.
  • Frameworks and libraries such as Tkinter, Django, and Flask can be used as per the problem statement requirements.

    • Pre-built Code:
  • Reusing small code snippets or templates is allowed if credited appropriately.
  • Projects must not include entirely pre-built solutions.

    • Version Control:
  • Teams are encouraged to use version control platforms like GitHub to manage their code.

Disqualification Rules

    Plagiarism:

You must bring your own laptop, charger, and any additional accessories required for development.
